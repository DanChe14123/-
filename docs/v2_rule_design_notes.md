# v2 规则设计说明

## 1. 设计目标

`v2.0` 的目标是把原有安全教育原型升级为中量协作桌游。

本次重构不改变主题，不改成剧本杀、社交推理或电子游戏，而是强化桌游层面的持续决策：

- 从单次事件处置升级为风险状态管理
- 从团队公共讨论升级为岗位协作
- 从共同胜负升级为共同胜负加个人贡献
- 从随机任务升级为阶段化实验路线

## 2. v1.0 的保留内容

继续保留：

- 化学实验室安全培训定位
- 任务、事件、行动三类核心卡牌
- 事件背后的复盘知识点
- 教学、标准、挑战三种模式
- 可通过模拟器验证的数值设计思路

## 3. v1.0 的替换内容

替换或重写：

- 一次性正确/错误判定
- 过于统一的公共手牌决策
- 较弱的角色差异
- 较弱的局内长期压力
- 说明书与评委说明混杂的问题

## 4. 新机制的设计理由

### 岗位轮值

实验室安全不是单人答题，而是多人协作。岗位轮值让玩家在不同轮次承担不同责任，从而提升参与感和教学真实性。

### 风险状态

真实实验风险通常不会立刻爆发，而是从苗头到暴露再到失控。状态机制让玩家理解“早发现、早处理”的价值。

### 交接链

实验室工作常常存在交接问题。交接链把上一轮未处理干净的风险带到下一轮，使玩家理解记录、交接和复盘的重要性。

### 个人贡献

纯协作游戏容易出现少数玩家主导。贡献分不改变团队胜负，但能鼓励每名玩家承担岗位职责。

### 任务路线

阶段化路线让一局游戏更像真实实验流程：准备、操作、收尾、应急，而不是完全随机事件拼接。

## 5. 本轮不做的内容

- 不引入隐藏身份
- 不做阵营对抗
- 不增加主持人强依赖
- 不让个人贡献变成输赢核心
- 不在规则未稳定前全量重做卡面

## 6. 需要后续验证的问题

- 岗位能力是否过强
- 贡献分是否增加记账负担
- 风险状态是否让游戏过难
- 阶段化任务是否降低重玩性
- 行动牌数量是否足以支撑 20-30 分钟体验
