# 版本备份规范

## 1. 目的

本文件用于约束后续每个版本的备份方式，避免规则、模拟器和数据脱节。

## 2. 每个正式版本必须保留的内容

每当出现一个新的正式版本，例如 `v0.4`，必须同时保留以下内容：

1. 规则文件
2. 模拟器脚本
3. 正式跑数的数据目录
4. 测试规程
5. 在 `CHANGELOG.md` 中增加一条版本记录

## 3. 推荐目录结构

- 规则：`versions/vX.Y/boardgame_rules.md`
- 模拟器：`versions/vX.Y/run_balance_sim.py`
- 正式数据：`data/vX.Y/run_XXX_fullstats/`
- 测试规程：`docs/ai_balance_test_protocol_vX.Y.md`

## 4. 数据目录命名规则

建议统一使用：

- 烟测：`smoke_test_*`
- 正式数据：`run_XXX_*_fullstats`

示例：

- `run_001_v0_3_fullstats`
- `run_002_v0_4_fullstats`

## 5. 当前工作方式

默认情况下，不需要每次对话都重新阅读旧版本全量文件。

后续工作时优先依赖：

- `CHANGELOG.md`
- `docs/version_memory.md`

只有在以下情况才回看旧版本原文：

- 准备复赛汇报
- 写正式报告
- 比较两个版本的具体机制差异
- 需要复盘某次模拟数据的来源

## 6. 禁止事项

- 不要覆盖旧版本正式数据
- 不要直接修改旧版本归档文件
- 不要只保留结论，不保留原始数据
- 不要出现“规则版本”和“模拟器版本”不一致但未记录的情况
