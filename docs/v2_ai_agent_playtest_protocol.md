# v2 AI 代理试玩测试方案

## 1. 目的

### Writing Guidance

本方案用于在暂时没有真人试玩条件时，先用 AI 代理发现规则说明、回合流程和教学表达中的问题。它不能替代真人试玩，也不能证明真实玩家体验必然良好。

本轮 AI 代理测试只回答 4 个问题：

- 新手是否能看懂怎么开局。
- 每轮流程是否有歧义。
- 事件、行动、岗位之间的关系是否容易误解。
- 评委或教师是否能理解教学价值。

## 2. 输入材料

给所有 AI 代理统一提供以下材料：

- `manual/player_rulebook_v2.md`
- `manual/quick_reference_v2.md`
- `manual/judge_brief_v2.md`
- `docs/v2_sample_playthrough_standard.md`
- `docs/v2_balance_report.md`

禁止 AI 代理假设已经有真人试玩结果。

## 3. 四类代理

| 代理 | 模拟对象 | 主要任务 |
| --- | --- | --- |
| A 新手玩家 | 不熟悉桌游的学生 | 找出开局和回合流程中的不懂之处 |
| B 桌游玩家 | 熟悉协作桌游的玩家 | 评价决策密度、节奏和是否有无聊回合 |
| C 化学院教师 | 安全教育使用者 | 评价知识点、课堂带玩和复盘价值 |
| D 规则裁判 | 规则执行者 | 查找规则歧义、字段缺失和结算冲突 |

## 4. 测试流程

### Step 1：独立阅读

每个代理先独立阅读输入材料，不互相讨论。

输出：

| 字段 | 要求 |
| --- | --- |
| 能否开局 | 用 1-5 分评分 |
| 最大疑问 | 写 3 条以内 |
| 必须补充的信息 | 写 3 条以内 |
| 认为最清楚的部分 | 写 2 条以内 |

### Step 2：规则复述

每个代理用自己的话复述：

- 游戏目标是什么。
- 每轮做什么。
- 玩家如何选择行动。
- 怎么赢，怎么输。
- 复盘牌有什么用。

若复述和规则书不一致，记录为“规则理解偏差”。

### Step 3：模拟讨论一轮

让 4 个代理共同模拟一轮：

1. 抽一个任务。
2. 抽一个事件。
3. 每个代理说自己建议的行动。
4. 规则裁判判断是否符合规则。
5. 化学院教师补充现实实验室解释。

输出争议点：

| 争议点 | 涉及规则 | 是否影响继续游玩 | 建议修订 |
| --- | --- | --- | --- |

### Step 4：评委视角审查

让 C 化学院教师和 D 规则裁判共同回答：

- 如果我是评委，3 分钟内能否理解作品是什么？
- 如果我是老师，能否直接拿去带一次课堂导入？
- 哪些内容更像桌游，哪些内容更像安全问答？
- 哪些说法可能被认为证据不足？

## 5. 记录格式

每次 AI 代理测试输出一个 Markdown 文件，建议命名：

```text
data/v2/ai_agent_playtests/agent_playtest_YYYYMMDD_round01.md
```

固定结构：

```markdown
# v2 AI 代理试玩记录

## 基本信息
- 日期：
- 使用模型：
- 输入材料：
- 是否使用真人玩家：否

## 四类代理独立反馈

## 规则复述偏差

## 模拟讨论记录

## 主要问题排序

## 建议修改

## 不能作为事实的内容
- 本记录不能证明真人玩家体验。
- 本记录不能替代课堂试用。
```

## 6. 评分规则

| 分数 | 含义 |
| ---: | --- |
| 5 | 几乎不需要解释即可理解 |
| 4 | 有少量术语需要确认 |
| 3 | 能玩，但需要主持人辅助 |
| 2 | 多处流程无法独立判断 |
| 1 | 无法开局或无法结算 |

验收目标：

- 新手玩家代理“能否开局”不低于 3。
- 规则裁判代理不应发现阻断性冲突。
- 化学院教师代理能说清至少 3 个教学价值点。
- 桌游玩家代理能指出至少 2 个真实决策点。

## 7. 复制给低级 AI 的完整指令

```text
你将模拟一次《实验室安全值班》v2 的无真人试玩替代测试。请严格区分“规则文件中明确写了什么”和“你作为代理的推测”。不要声称已经有真人试玩。

请扮演四个代理：
1. 新手玩家：不熟悉桌游，只关心能不能看懂怎么开始。
2. 桌游玩家：熟悉协作桌游，关心节奏、决策和可玩性。
3. 化学院教师：关心实验室安全教学价值。
4. 规则裁判：关心规则是否能执行、是否有歧义。

请阅读以下材料：
- manual/player_rulebook_v2.md
- manual/quick_reference_v2.md
- manual/judge_brief_v2.md
- docs/v2_sample_playthrough_standard.md
- docs/v2_balance_report.md

输出必须包含：
1. 四类代理各自的开局理解评分，1-5 分。
2. 每类代理最多 3 个疑问。
3. 每类代理用自己的话复述游戏目标、每轮流程、胜负条件。
4. 至少 1 轮模拟讨论，写出任务、事件、行动建议和争议点。
5. 按严重程度列出最多 8 个问题。
6. 明确写出：本测试不是真人试玩，不能证明真实玩家体验。
```
